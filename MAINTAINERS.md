### Note (pre-alpha)

While the project is pre-alpha, there is one maintainer (the BDFL).
Listing a personal contact address here would rot fast and invites
spam. Until the project domain is stood up:

- **Security reports** → GitHub Security Advisories on the repo
  (see [SECURITY.md](./SECURITY.md)). Do not post security reports in
  public issues.
- **Conduct reports** → DM any maintainer on the project chat (links
  in the README) until `conduct@valenx.org` is live.
- **Everything else** → GitHub Issues / Discussions.

As soon as the domain and project email are live, those addresses go
here and the placeholders above get replaced.

## Becoming a maintainer

See [CONTRIBUTING.md → Becoming a maintainer](./CONTRIBUTING.md#becoming-a-maintainer).
Short version: 5+ substantial PRs, 6+ months of sustained engagement,
constructive reviews, invited by current-maintainer consensus.

## How this file stays accurate

- New maintainer: added in the PR that grants them access
- Role change or step-down: their own PR, or any other maintainer's PR
  with their written consent (linked in the PR description)
- Every file update is a normal PR through the usual review flow
