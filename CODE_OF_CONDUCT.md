# Code of Conduct

Valenx adopts the **Contributor Covenant, version 2.1** as its code of
conduct. The full text is published by the Contributor Covenant project
and is incorporated here by reference:

<https://www.contributor-covenant.org/version/2/1/code_of_conduct/>

The short version: we expect everyone participating in this project —
contributors, maintainers, users on any of our discussion channels — to
be respectful, welcoming, and professional. Harassment, insults, or
demeaning behavior are not tolerated.

## Scope

This code applies to all project spaces: the GitHub repository (issues,
PRs, discussions, code comments), official chat channels (Matrix,
Discord, mailing lists), and any in-person or virtual event held under
the Valenx name.

It also applies when an individual is officially representing the
project in public spaces — for example, speaking at a conference on
behalf of Valenx, or posting from a project social-media account.

## Reporting

If you experience or witness behavior that violates this code, please
report it to the maintainers:

- **Email:** `conduct@valenx.org` (set up in Year 1)
- **Direct message** to any listed maintainer on Matrix / Discord

All reports will be reviewed and investigated promptly. Reporters'
identities will be kept confidential; reporters will not be retaliated
against.

## Enforcement

Maintainers are responsible for clarifying and enforcing these
standards. They may take any action they deem appropriate, including
but not limited to:

1. **Warning** — a private or public note that specific behavior was out of line
2. **Temporary ban** — from a specific channel, PR, or the project for a defined period
3. **Permanent ban** — from all project spaces

Enforcement decisions follow the Contributor Covenant's
[Enforcement Guidelines](https://www.contributor-covenant.org/version/2/1/code_of_conduct/#enforcement-guidelines),
which ladder responses proportionally from correction → warning →
temporary ban → permanent ban.

## Maintainer accountability

Maintainers are held to the same standard as everyone else — and to a
higher standard in terms of professionalism when acting officially.
Reports about a maintainer are reviewed by the rest of the maintainer
group (or, in early years, the BDFL plus one other trusted
contributor) to avoid conflicts of interest.
